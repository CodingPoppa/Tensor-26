How This Integration Works

Our AI scanner backend runs on your laptop (FastAPI), not on a cloud host like Railway or Render. GitHub Actions runs in the cloud, so it cannot reach your local machine directly. Ngrok creates a secure public URL that tunnels back to your local backend so the CI pipeline can call /api/health and /api/scan safely during each pull request.

Step-by-Step Setup

  1. Start the backend locally. Run your FastAPI app on port 8000 (for example: uvicorn main:app --reload --port 8000).
  2. Expose it with ngrok. In another terminal, run ngrok http 8000. Ngrok will give you an HTTPS URL like https://something.ngrok-free.dev that points to your local FastAPI server.
  3. Paste the ngrok URL below. Copy the HTTPS URL from the ngrok terminal and paste it into the “Ngrok URL” input on this page.
  4. Test the connection. Click “Test Connection”. We call GET /api/health through ngrok and expect {"status":"ok"}. If it works, the status turns green (“Connected”).
  5. Generate the GitHub Action. As soon as the URL is entered, the YAML on the right is generated for you, with your ngrok URL wired into the curl call to /api/scan.
  6. Install the workflow into your repo. Click “Copy Code”, then in your target repository create .github/workflows/security.yml and paste the YAML there. Commit and push.
  7. Open a Pull Request. When you open or update a PR, GitHub Actions will: compute the git diff, send it as JSON to your scanner at /api/scan via ngrok, and use the response to decide whether to pass or fail the security check and post a PR comment with the findings.

Why the backend runs locally

For this hackathon, we are not deploying the backend to a permanent cloud service (Railway, Render, etc.). Instead, the scanner runs on your machine as a FastAPI app, and ngrok temporarily exposes it to the internet. This keeps deployment simple while still giving a realistic CI/CD integration where GitHub Actions calls your AI scanner in real time during each pull request.
